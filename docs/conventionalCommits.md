# Commit Message Conventions

Use the following keywords to categorize your commits for better project tracking and readability:

| Keyword | Description |
| :--- | :--- |
| **feat** | A new feature or completed task |
| **fix** | A bug fix |
| **refactor** | Code restructured but behavior unchanged |
| **chore** | Setup, config, tooling, renaming files |
| **docs** | Documentation, README, report writing |
| **style** | Formatting, naming, no logic change |
| **test** | Adding or updating tests |
| **ci** | GitHub Actions, pipelines |
| **db** | Schema changes, migrations, seed data |

## Example Usage
`feat: add user authentication module`  
`fix: resolve memory leak in data parser`
